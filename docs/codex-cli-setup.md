# Codex CLI Setup

This document describes the Codex CLI customization added on June 5, 2026.
The configuration focuses on session visibility, usage limits, token totals,
project context, and faster terminal workflows.

## Files Changed

- `~/.codex/config.toml`: Codex terminal UI configuration.
- `~/.zshrc`: Codex completion and command aliases.

The installed Codex version was `0.137.0` when this setup was configured.

## Status Line

The following `[tui]` configuration was added to `~/.codex/config.toml`:

```toml
[tui]
alternate_screen = "never"
notifications = true
show_tooltips = true
status_line = [
  "model-with-reasoning",
  "fast-mode",
  "current-dir",
  "git-branch",
  "context-remaining",
  "used-tokens",
  "total-input-tokens",
  "total-output-tokens",
  "five-hour-limit",
  "weekly-limit",
  "codex-version",
  "thread-id",
]
terminal_title = ["activity", "project-name", "git-branch"]
```

## Why These Fields Were Selected

- `model-with-reasoning`: confirms the active model and reasoning level.
- `fast-mode`: shows whether the performance mode is active.
- `current-dir`: confirms the active working directory.
- `git-branch`: helps prevent changes on the wrong branch.
- `context-remaining`: shows how much context remains in the active session.
- `used-tokens`: shows total token use for the current Codex thread.
- `total-input-tokens`: shows tokens sent to the model.
- `total-output-tokens`: shows tokens generated by the model.
- `five-hour-limit`: shows short-term account usage when available.
- `weekly-limit`: shows weekly account usage when available.
- `codex-version`: helps diagnose version-specific behavior.
- `thread-id`: identifies the session when resuming or troubleshooting.

Codex does not currently expose account-lifetime usage as a status line field.
The displayed token total applies to the current thread.

## Additional TUI Behavior

`alternate_screen = "never"` preserves terminal scrollback after Codex exits.

Notifications remain enabled so long-running tasks can signal completion.
Tooltips remain enabled to make Codex commands and controls easier to discover.

The terminal title displays:

- Current Codex activity.
- Project name.
- Git branch.

## Completion

Codex Zsh completion is generated and loaded during shell startup with:

```sh
codex completion zsh
```

This enables Tab completion for Codex commands and options.

## Command Aliases

| Alias | Command | Purpose |
| --- | --- | --- |
| `c` | `codex` | Start Codex |
| `cr` | `codex resume --last` | Resume the latest session |
| `cf` | `codex fork --last` | Fork the latest session |
| `cx` | `codex --no-alt-screen` | Start Codex with terminal scrollback |
| `cdoctor` | `codex doctor` | Diagnose the Codex installation |
| `cfeatures` | `codex features list` | List available Codex features |

## Installation

The Codex setup was installed by:

1. Inspecting `~/.codex/config.toml` and the installed Codex CLI.
2. Checking Codex version `0.137.0`.
3. Verifying the status line identifiers supported by the installed CLI.
4. Adding the `[tui]` section directly to `~/.codex/config.toml`.
5. Adding completion and aliases directly to `~/.zshrc`.
6. Validating the configuration with:

   ```sh
   codex --strict-config doctor --summary
   ```

The strict configuration check passed. The doctor report confirmed that the
local installation, authentication, configuration, state databases, Git
integration, and Ghostty terminal detection were valid.

The doctor report also detected restricted network reachability in the
execution environment. This was separate from the local configuration.
